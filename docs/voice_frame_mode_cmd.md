# 语音帧模式配置命令

## 命令码

| 命令 | 命令码 | 说明 |
|------|--------|------|
| SET_VOICE_FRAME_MODE | `0x70` | 设置上行语音帧模式 |
| GET_VOICE_FRAME_MODE | `0x71` | 查询当前上行语音帧模式 |

---

## SET_VOICE_FRAME_MODE (0x70)

**设置设备上行语音编码的帧聚合模式**，影响每次 `AT^AUDPCM` 命令中包含的 AMR-NB 帧数量。

### 请求格式

```
[seq] [0x70] [param_len=1] [frames] [crc16_hi] [crc16_lo]
```

| 字段 | 偏移 | 长度 | 说明 |
|------|------|------|------|
| seq | 0 | 1 | 序列号 |
| cmd | 1 | 1 | `0x70` |
| param_len | 2 | 1 | 固定 `0x01` |
| frames | 3 | 1 | 帧数：`0x01` 或 `0x03` |
| crc16 | 4 | 2 | CRC16-CCITT |

### 参数值

| 值 | 含义 | 对应平台 |
|----|------|----------|
| `0x01` | 1 帧/包（20ms） | Android |
| `0x03` | 3 帧/包（60ms） | iPhone |

### 响应格式

```
[seq] [0x70] [resp_code] [data_len=1] [current_mode] [crc16_hi] [crc16_lo]
```

| 字段 | 偏移 | 长度 | 说明 |
|------|------|------|------|
| seq | 0 | 1 | 回显序列号 |
| cmd | 1 | 1 | `0x70` |
| resp_code | 2 | 1 | `0x00`=成功, `0x03`=参数无效 |
| data_len | 3 | 1 | `0x01` |
| current_mode | 4 | 1 | 当前生效的帧模式 |
| crc16 | 5 | 2 | CRC16-CCITT |

---

## GET_VOICE_FRAME_MODE (0x71)

**查询当前上行语音帧模式。**

### 请求格式

```
[seq] [0x71] [param_len=0] [crc16_hi] [crc16_lo]
```

### 响应格式

```
[seq] [0x71] [resp_code] [data_len=1] [current_mode] [crc16_hi] [crc16_lo]
```

| 字段 | 说明 |
|------|------|
| resp_code | `0x00`=成功 |
| current_mode | `0x01` 或 `0x03` |

---

## 行为说明

1. **默认值**：设备上电默认 `1` 帧/包模式
2. **手动锁定**：通过 `0x70` 设置后进入手动模式，下行数据的自动帧数检测被禁用，不会因收到不同帧数的下行包而改变上行模式
3. **切换时机**：可在通话前或通话中随时切换
4. **编码行为**：
   - 模式 1：每收到 1 帧 PCM（20ms）立即编码发送一个 `AT^AUDPCM` 包
   - 模式 3：累积 3 帧 PCM（60ms）后编码发送一个 `AT^AUDPCM` 包

## 示例

**设置 3 帧模式（iPhone）：**
```
发送: 0x01 0x70 0x01 0x03 [crc16]
响应: 0x01 0x70 0x00 0x01 0x03 [crc16]
```

**查询当前模式：**
```
发送: 0x02 0x71 0x00 [crc16]
响应: 0x02 0x71 0x00 0x01 0x03 [crc16]
```

**传入无效值：**
```
发送: 0x03 0x70 0x01 0x02 [crc16]
响应: 0x03 0x70 0x03 0x00 [crc16]    // resp_code=0x03 参数无效
```
